UserPreferenceService
service in module userPreference

Description

This service contains common CRUD operations for user preferences using userPreference.service:UserPreferenceFactory

Properties

  • updatePreference(preferenceName,newValue)

    promise chain to update and get the new value for a given preference

  • saveResponseDataToCache(responseData)

    given a response data from the user preference api it will retrieve the preference value and save it to the stored cache

  • getPreference(preferenceName)

    given a preference name it will retrieve the stored cache if it exists in the cache, else it will retrieve an empty hash

  • hasPreference(preferenceName)

    given a preference name it will check if the key exists in the cache.

  • populateCache()

    will populate a cache from the ReST apis given the hash of preference names in userPreference.object:UserPreferenceKeys
