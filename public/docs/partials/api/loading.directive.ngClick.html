ngClick
directive in module loading

Description

This directive is used to display loading animations in a button or elsewhere. It will also add in necessary dom when specified within the given element.

In the case where this is not a route change request you must broadcast/emit the event depending on the locations of the loading element and controller using the ppdLoadingActionFinish event.

Usage

as attribute
<ANY ng-click="{function}"
     [loading="{boolean}"]
     [animation-target="{string}"]
     [animation-class="{string}"]
     [loading-style="{string}"]
     [disable-clicks="{boolean}"]
     [create-child-element="{boolean}"]
     [done-callback="{function}"]
     [prevent-bubbling="{boolean}"]>
   ...
</ANY>

Parameters

ParamTypeDetails
ngClickfunction

option must contain a scope method callback for when this is clicked. It should usually be a route change request.

loading
(optional)
boolean

option to enable/disable loading animations on this element

(default: false)

animation-target
(optional)
string

option must be given a valid selector (preferably an id) to send the animation classes to

(default: self)

animation-class
(optional)
string

class to apply when loading

(default: loading)

loading-style
(optional)
string

class to apply on top of the animation class

(default: null)

disable-clicks
(optional)
boolean

flag to disable click of all other ppdLoadingClick directives

(default: false)

create-child-element
(optional)
boolean

flag to add a child element to itself of 

<span class='loading-spinner'></span>

(default: false)

done-callback
(optional)
function

callback to be triggered when expression in ppd-loading-click has completed

(default: function(){})

prevent-bubbling
(optional)
boolean

flag to enable/disable stopPropagation on click event

(default: false)
